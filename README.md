# IronClaw Go

IronClaw 的混合 Go + Rust 架构 —— 安全的个人 AI 助手。

## 架构

```
┌─────────────────────────────────────────────────────────┐
│                    Go 编排器 (Orchestrator)               │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌──────────────┐  │
│  │ Web     │ │ CLI/    │ │ 配置    │ │  数据库      │  │
│  │ 网关    │ │ REPL    │ │ 管理器  │ │ (PG/libSQL)  │  │
│  │ (未来)  │ │         │ │         │ │              │  │
│  └────┬────┘ └────┬────┘ └────┬────┘ └──────┬───────┘  │
│       └────────────┴───────────┴─────────────┘          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌──────────────┐  │
│  │ 工具    │ │ Agent   │ │ 通道    │ │ gRPC 客户端  │  │
│  │ 注册表  │ │ 循环    │ │ 管理器  │ │ (wasm)       │  │
│  └─────────┘ └─────────┘ └─────────┘ └──────────────┘  │
└─────────────────────────────────────────────────────────┘
                           │ gRPC (Unix 域套接字)
┌─────────────────────────────────────────────────────────┐
│              Rust WASM 运行时 Sidecar                    │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌──────────────┐  │
│  │ wasmtime│ │WASM 工具│ │WASM 通道│ │  凭证注入器  │  │
│  │ 引擎    │ │ 执行器  │ │ 适配器  │ │              │  │
│  └─────────┘ └─────────┘ └─────────┘ └──────────────┘  │
└─────────────────────────────────────────────────────────┘
```

## 项目结构

| 路径 | 语言 | 用途 |
|------|------|------|
| `cmd/ironclaw/` | Go | 主编排器入口 |
| `internal/` | Go | 核心包（agent、channels、tools、db、config） |
| `proto/` | Protobuf | Go 与 Rust 之间的 gRPC 契约 |
| `wasm-runtime/` | Rust | WASM sidecar（wasmtime + WASI） |

## 快速开始

### 前置要求

- Go 1.22+
- Rust 1.78+
- `protoc`（Protocol Buffers 编译器）

### 构建

```bash
# 同时构建两边
make build

# 或分别构建
make build-go
make build-rust
```

### 运行

```bash
# 先启动 Rust sidecar
make run-rust

# 在另一个终端中，启动 Go 编排器
make run-go
```

### 生成 gRPC 代码

```bash
make proto
```

## 开发

```bash
make fmt      # 格式化 Go 和 Rust 代码
make test     # 运行两边的测试
make clean    # 清理构建产物
```

## 设计决策

- **Go 负责编排**：编译速度快，HTTP/DB 生态完善，并发简单易用。
- **Rust 负责 WASM**：wasmtime 是唯一生产级的 Component Model 运行时。
- **gRPC over UDS**：两个进程之间强类型、低延迟的进程间通信。
